{# ── E-Mail (SMTP) ──────────────────────────────────────────────────── #}

E-Mail-Versand (SMTP)

Konfiguration für Passwort-Reset-Mails und andere Benachrichtigungen. Leer lassen = Wert aus Umgebungsvariable (SMTP_*) verwenden.

EinstellungWertBeschreibung
smtp_host Hostname des Mailservers (z. B. smtp.gmail.com)
smtp_port TCP-Port — 587 (STARTTLS) oder 465 (SSL/TLS)
smtp_user Benutzername für die SMTP-Authentifizierung
smtp_password Passwort für die SMTP-Authentifizierung
smtp_from Absenderadresse (From-Header). Leer = smtp_user verwenden.
smtp_starttls STARTTLS-Verschlüsselung aktivieren (empfohlen für Port 587)
smtp_timeout Verbindungs-Timeout in Sekunden (Standard: 15)
{# ── Web Push-Benachrichtigungen ─────────────────────────────────────── #}

Web Push-Benachrichtigungen

Browser-Push bei neuen Einsätzen. Benötigt ein VAPID-Schlüsselpaar (web-push generate-vapid-keys). Leer lassen = Wert aus Umgebungsvariable (VAPID_*) verwenden.

EinstellungWertBeschreibung
enable_push Push-Benachrichtigungen systemweit ein- oder ausschalten
vapid_public_key Öffentlicher VAPID-Schlüssel — wird an Browser-Clients übermittelt
vapid_private_key Privater VAPID-Schlüssel — niemals öffentlich teilen
vapid_email Kontakt-E-Mail für Push-Dienst (VAPID-Claim, ohne mailto:)
{# ── FCM – Native Android Push ───────────────────────────────────────── #}

FCM – Native Android Push

Firebase Cloud Messaging für die native Android-App (einsatzleiter.cloud). Leer lassen = Wert aus Umgebungsvariable (FCM_*) verwenden. Erfordert ein Firebase-Projekt und ein Service-Account-JSON auf dem Server. Änderungen sind erst nach einem App-Neustart wirksam.

EinstellungWertBeschreibung
fcm_enabled FCM-Sendepfad für Android-App aktivieren
fcm_project_id Firebase-Projekt-ID (aus Firebase-Konsole → Projekteinstellungen)
fcm_credentials_path Absoluter Pfad zum Service-Account-JSON auf dem Server
{# ── Einsatz-Auto-Schließen ──────────────────────────────────────────── #}

Einsatz-Auto-Schließen

Offene Einsätze werden nach Ablauf der konfigurierten Zeit automatisch geschlossen. Benutzer können das über die „Offen halten"-Schaltfläche im Einsatz verzögern (setzt den Timer zurück).

EinstellungWertBeschreibung
incident_autoclose_enabled Auto-Schließen aktivieren oder deaktivieren (Standard: aktiv)
incident_autoclose_after_hours Stunden nach Einsatzbeginn bis zur „Noch offen?"-Warnung (Standard: 48 h)
incident_autoclose_grace_minutes Minuten zwischen der Warnung und dem tatsächlichen Schließen (Standard: 60 min)
