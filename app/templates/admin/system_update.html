Release-ZIP hochladen

Lade eine .zip-Datei hoch, die als Release verpackt wurde. Das ZIP muss mindestens ein app/-Verzeichnis und eine pyproject.toml enthalten.

Geschützte Dateien (werden nie überschrieben):
  • .env – Konfiguration und Secrets
  • alembic/versions/ – eigene Datenbank-Migrationen
  • app/static/img/uploads/ – hochgeladene Logos
Abbrechen

Hinweis zum Update-Prozess

  1. ZIP wird serverseitig validiert (muss app/ und pyproject.toml enthalten)
  2. Dateien werden in ein temporäres Verzeichnis extrahiert
  3. Alle nicht-geschützten Dateien werden über die bestehende Installation kopiert
  4. alembic upgrade head wird ausgeführt (Datenbank-Migrationen)
  5. Gunicorn erhält SIGHUP (graceful reload – laufende Requests werden abgeschlossen)
  6. Falls kein PID-File vorhanden: systemctl restart einsatzleiter

Empfehlung: Vor einem Update ein Datenbank-Backup erstellen.
